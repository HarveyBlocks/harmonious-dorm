# 和睦寝室（HarmoniousDorm）  校园宿舍生活可视化协调平台

## 一、 选题可行性分析

### 1. 贴近大学生的校园生活

宿舍矛盾和生活琐事是大学生活中最普遍、最痛点的场景之一。作息不一致、卫生值日遗忘、水电费分摊计算麻烦、公共物品使用冲突等问题，是每个宿舍都会面临的实际问题。该选题直接切入这些高频生活场景，用户需求真实且强烈。

### 2. 拥有符合人机交互特点的界面

此课题天然适合进行交互设计创新：

可视化日程：将枯燥的值日表转化为可视化的拖拽日历或时间轴。

状态感知：设计直观的“宿舍状态切换”（如：考试周模式、睡眠模式、派对模式），通过颜色变化反馈宿舍环境。

互动反馈：设计温和的提醒机制（如：不是生硬的通知，而是趣味性的“叫醒”动画），体现人机交互中的“情感化设计”。

### 3. 功能点少且易于实现

无需复杂的算法，核心是数据的增删改查（CRUD）与前端展示。

后端：Node.js仅需提供用户管理、宿舍加入、值日记录、账单计算等简单接口，配合轻量级SQLite数据库即可快速实现。

前端：Vue框架非常适合开发响应式的仪表盘和表单组件。

### 4. 选题新颖性分析

目前市面上的宿舍类应用多为“报修系统”（面向学校管理方）或“二手交易”。面向宿舍内部人际关系与生活协作的Web应用相对较少。通过引入“游戏化”或“可视化”的协调机制，可以很好地避免同质化。

## 二、 针对课程报告模版的具体内容设计

以下是该课题在报告各章节的撰写思路：

### 2.1 用户、场景、任务描述

用户画像：

典型用户A（焦虑的舍长）：责任心强，总是记得倒垃圾，但不好意思每次都催室友，希望能有一个客观的工具代替口头催促。

典型用户B（迷糊的室友）：经常忘记这周轮到谁值日，或者忘记交电费，需要清晰的提醒。

场景剧本：

场景：周五晚上22:00，宿舍比较吵闹，灯光昏暗。

任务：月底结算水电费，舍长需要计算每人平摊多少钱，还要扣除某位室友请假不在宿舍的天数。

痛点：使用计算器算半天，发微信群容易被刷屏，有人没看到导致收钱困难。情绪：烦躁、尴尬。

### 2.2 竞品描述和评价

竞品：微信群公告、Excel表格、记事本。

劣势：

微信群信息由于缺乏结构化展示，容易被覆盖，无法直观看到“谁没做”。

Excel表格在手机端操作体验极差，不符合移动端的人机交互原则（费茨定律，按钮太小）。

本作品优势：提供Web端大屏查看与移动端适配，数据可视化，一键提醒，降低认知负荷。

### 2.3 设想新的任务逻辑 & 2.5 详细设计交互阶段

针对该课题，建议设计以下三个核心功能模块，体现交互设计过程：

功能一：可视化值日大转盘（解决卫生协调）

交互设计：

界面不再是枯燥的表格，而是一个圆形的转盘或时间轴。

操作：舍长拖拽头像到对应的日期，系统自动生成值班提醒。

反馈：当值日生完成值日上传照片后，头像亮起绿色光环；未完成则显示灰色并有轻微抖动提示。

理论依据：利用可见性原则，让宿舍卫生状态一目了然。

功能二：一键AA账单计算器（解决财务纠纷）

交互设计：

输入总金额 -> 勾选参与分摊的室友 -> 自动计算人均金额。

界面设计模拟“收据”样式，增加真实感。

点击“发送催款”按钮，生成一张带有金额和二维码的卡片（模拟微信红包样式），方便转发。

功能三：宿舍状态同步器（解决作息冲突）

交互设计：

设计一个“门牌”样式的虚拟组件。

成员可以点击切换自己的状态：“在学习”、“在睡觉”、“在游戏”、“不在寝室”。

当有两人以上点击“在睡觉”时，宿舍主页背景自动切换为深色模式（夜间模式），并提示其他室友保持安静。

理论依据：情境感知与约束设计，通过环境暗示引导用户行为，减少言语冲突。



## 三、 总结

“宿舍生活协调”是一个极佳的课程设计选题。它既避免了复杂的业务逻辑干扰设计重点，又拥有丰富的用户痛点可供挖掘。

优势总结：

交互空间大：可以尽情施展UI设计才华（图标、动效、状态切换）。

数据结构简单：后端开发压力小，可以集中精力在前端交互体验的打磨上。

实用性强：开发完成后甚至可以直接在自己的宿舍试用，进行真实的“可用性评估”。
